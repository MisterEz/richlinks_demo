# Compiling

Compiling with gulp

Compiling is only needed to make changes visible. This project does not need to be compiled to view.

Install node.js & npm
  • Linux: $ sudo apt install nodejs npm OR
  • Other: https://nodejs.org/en/
Install project dependencies
  • $ cd [project_path]
  • $ npm install
Install gulp globally
  • $ npm install gulp-cli -g
Run Gulp
  • $ gulp
Project directory /public/ should now contain the updated web files.
Not required to view

# Technical Information

Dependencies

This project uses the following:
  • Gulp
  • Handlebars
  • jQuery 3
  • Sass
  • Bootstrap for Sass 3 (Modified)

Gulp is the task runner for compiling the project. (See #compiling for usage).

Handlebars is a template engine used for shared html components.

jQuery is a javascript convenience library. jQuery Docs

Sass is a popular CSS preprocessor. Syntax is very similar to CSS but developers should make them self familiar with it before working on the project. Sass docs.

Bootstrap is a css framework. This project uses a custom build so some features you are familiar with may not be avaliable. For a full list of compiled components see /resources/sass/_boostrap-custom.scss

The main bootstrap components included are:
  • Grid
  • Typography
  • Buttons
  • Navbar
  • Code

File Structure

The main project structure is show below.

/
    /resources/ - Pre-compiled files (Sass, HTML partials)
        /sass/
        /html/
    /public/    - Compiled files. (WEB ROOT)
        /img/
        /css/
        /js/
        /style_guide/

Performance

For purposes of assesment, files will not be minified. (With exception of compiled .css)

# Accessibility

Responsive Design

Desktop First Design. Given this website is intended for a technical audience, prioritze desktop views first. Mobile views should still be considered but it is acceptable for compromises to be made on smaller screen sizes.

Optimise for 1366x768 px. As smaller screen sizes make up ~50% of desktop web users. Website made for smaller screen look _alright_ at higher resolutions but vice versa doesn't work at all.

Bootstrap provides the .container class for limiting screen size to 1170px width.
Avoid using .container-fluid

Browser Support

IE / Edge Chrome Firefox Safari
9+ (yes) (yes) (yes)

Check https://caniuse.com for feature compatability. As a rule of thumb aim for 95%+ Global prefixed support.

Accessibility guidelines

  • Ensure HTML is valid (With exception to third-party iframes)
  • Use html navigational aids
    • <Header>
    • <Aside>
    • <Nav>
    • <Main>
    • etc...
  • Implement ARIA Enhancements (Docs)
  • Do not misuse <h[1-6]> tags.
  • Add skip to main content link (See Below)
Skip to main content HTML snippit: (From W3C) 
<div id="skiptocontent">
        <a href="#maincontent">skip to main content</a>
    </div>
